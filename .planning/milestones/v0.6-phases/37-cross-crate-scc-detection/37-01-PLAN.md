<objective>
Extend CallGraph to inject cross-crate edges from ContractDatabase so Tarjan's SCC algorithm can form strongly connected components that span crate boundaries.

Purpose: XCREC-01 requires that `cargo verify` detect mutually recursive call cycles between functions in different crates. The current `CallGraph::from_functions` only knows about in-crate call edges. We add a new constructor `from_functions_with_cross_crate_db` that also adds edges for any contracted callee (from the DB) that the in-crate functions call, making those callees "virtual nodes" in the graph. If a virtual callee transitively calls back into an in-crate function (as recorded by the callee's own DB entry having an in-crate function as a callee), Tarjan's will detect the cross-crate SCC.

Output: New public constructor on `CallGraph`. All existing tests unchanged. New TDD tests: cross-crate SCC detected, non-recursive cross-crate call not mistakenly flagged, normalize_func_name handles fully-qualified cross-crate paths.
</objective>

**Key design insight for cross-crate SCC:**

The DB does NOT know what functions a contracted callee calls. To detect a cross-crate SCC we need to know: "does callee_crate::bar call back into local::foo?"

Strategy: We use the `ContractDatabase` only to inject **one-way edges** from in-crate callers to the contracted callee node. For detecting the **back-edge** (callee calls back into in-crate function), we rely on the fact that `generate_termination_vcs` already receives the callee `decreases` via the DB — but for SCC detection itself, the back-edge cannot be inferred from the DB alone.

**Revised simpler design (KISS):**

A cross-crate SCC is detectable IF the cross-crate callee also appears as a contracted function in the DB AND its `contracts.decreases` is set AND the in-crate function calls it AND the in-crate function is also called by something that calls the cross-crate function. This is complex.

**Practical design for this phase:**

Represent cross-crate callees as virtual nodes. Add them to `all_functions`. Add edges: `in_crate_caller -> cross_crate_callee` (from `Terminator::Call` in MIR). For back-edges (`cross_crate_callee -> in_crate_fn`): scan DB entry for each contracted callee's `param_names` to find if any `param_names` matches an in-crate function name — this is too weak.

**Correct approach:** The DB entry for a function records its contracts, not its callees. Cross-crate SCC detection requires the dependency crate's call graph, which is not available from DB alone.

**What we CAN do (scope-appropriate):** Accept a `cross_crate_edges: &[(String, String)]` parameter (caller_name -> callee_name pairs extracted from MIR Terminator::Call where the callee is in the DB). The driver or test will supply these edges explicitly. `from_functions_with_cross_crate_db` builds the standard graph + adds the DB functions as virtual nodes + adds the explicit cross-crate back-edges.

For the driver integration (Phase 37 end-to-end), the "cross-crate back-edge" is supplied by reading the dep-crate's DB entry's `decreases` annotation with a special convention: if a DB entry has `contracts.decreases` set AND the decreases expression references an in-crate local function by name, that signals a cross-crate SCC.

**Final scoped design:**
- `from_functions_with_cross_crate_db(functions, contract_db)`: for each in-crate function's `Terminator::Call` where the callee is in `contract_db`, add a virtual node + edge. For virtual nodes that have `contracts.decreases` that is non-None, check if the decreases expression text references an in-crate function name — if so, add reverse edge (callee -> in-crate caller), creating a potential SCC. This is heuristic but sufficient for the declared use cases.

    ```rust
    pub fn from_functions_with_cross_crate_db(
        functions: &[(String, &Function)],
        contract_db: &crate::contract_db::ContractDatabase,
    ) -> Self {
        // Start with the standard in-crate graph
        let mut cg = Self::from_functions(functions);

        // Collect in-crate function names (normalized) for back-edge detection
        let in_crate_norm: HashSet<String> = functions
            .iter()
            .map(|(n, _)| normalize_func_name(n))
            .collect();

        // For each in-crate function, find calls to contracted callees not already in graph
        for (_name, func) in functions {
            for bb in &func.basic_blocks {
                if let crate::ir::Terminator::Call { func: callee_name, .. } = &bb.terminator {
                    let norm_callee = normalize_func_name(callee_name);
                    // Skip if already an in-crate node
                    if cg.all_functions.contains(&norm_callee) {
                        continue;
                    }
                    // Check if contracted callee is in DB (full name lookup)
                    if let Some(summary) = contract_db.get(callee_name) {
                        // Add virtual node for the cross-crate callee
                        cg.all_functions.insert(norm_callee.clone());
                        cg.name_map.entry(norm_callee.clone()).or_insert_with(|| callee_name.clone());
                        // Edge: in-crate caller -> cross-crate callee (already in edges from from_functions)

                        // Back-edge detection: if DB entry's decreases references an in-crate fn name,
                        // add reverse edge (cross-crate callee -> in-crate fn) to form potential SCC
                        if let Some(dec) = &summary.contracts.decreases {
                            for in_fn_norm in &in_crate_norm {
                                if dec.raw.contains(in_fn_norm.as_str()) {
                                    cg.edges
                                        .entry(norm_callee.clone())
                                        .or_default()
                                        .push(in_fn_norm.clone());
                                }
                            }
                        }
                    }
                }
            }
        }
        cg
    }
    ```
