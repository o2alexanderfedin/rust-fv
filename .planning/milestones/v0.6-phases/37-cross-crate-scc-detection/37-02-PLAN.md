<objective>
Wire generate_termination_vcs to use ContractDatabase for cross-crate callee decreases measures, and update vcgen.rs to use from_functions_with_cross_crate_db.

Purpose: XCREC-02 requires that `cargo verify` checks `#[decreases]` termination measures across crate boundaries. Currently `generate_termination_vcs` ignores the `_contract_db` parameter. This plan activates it: when a recursive call site's callee is a cross-crate function (not in the local `func_list`), the function looks up the callee's `contracts.decreases` from the DB to know the callee's measure for the cross-boundary VC. vcgen.rs must also use `from_functions_with_cross_crate_db` so the recursive_groups include cross-crate SCCs (built in Plan 01).

Output: `recursion.rs::generate_termination_vcs` uses contract_db for cross-crate callees. `vcgen.rs` passes contract_db to the new CallGraph constructor. New TDD unit tests verify the cross-crate termination VC has the correct SMT structure.
</objective>

```rust
pub fn generate_termination_vcs(
    func: &Function,
    recursive_groups: &[RecursiveGroup],
    _contract_db: Option<&ContractDatabase>,   // currently unused — underscore prefix
) -> Vec<VerificationCondition> { ... }
```

The current implementation (lines 94-216 of recursion.rs):
- Scans `func.basic_blocks` for `Terminator::Call` where callee is in `group`
- `normalize_callee(callee_name)` — strips path prefix
- Builds substitution from `func.params` -> actual `args`
- The entry measure comes from `func.contracts.decreases`
- The call-site measure is the entry measure with args substituted

For cross-crate: the callee is in `group` (because plan 01 adds virtual nodes to the SCC), but the callee's own `Function` body is not available. The callee's decreases measure is in `contract_db.get(callee_name).contracts.decreases`. We need to generate VCs that assert: **callee's decreases measure (at call site) < local function's decreases measure (at entry)**. This requires substituting the callee's formal params with the actual call args.
