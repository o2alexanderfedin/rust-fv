# Phase 37.1: Inferred Summary + Alias Precondition Guard - Context

**Gathered:** 2026-03-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Guard the `is_inferred` early-continue at `vcgen.rs:2441` so that alias VCs are not silently dropped when `callee_summary.alias_preconditions` is non-empty on a callee that also has `is_inferred=true`. Emit a V06x-series diagnostic warning in that co-occurrence case. This is a targeted bug fix — no new user-visible features.

</domain>

<decisions>
## Implementation Decisions

### Guard logic
- The `is_inferred` early-continue (line 2441–2448) must be conditional: if `alias_preconditions` is non-empty, skip the early `continue` and fall through to alias VC generation
- When `is_inferred=true` AND `alias_preconditions` is empty: behavior is unchanged (continue, suppress OpaqueCallee)
- When `is_inferred=true` AND `alias_preconditions` is non-empty: emit alias VCs + a diagnostic warning

### Diagnostic
- Severity: **warning** (not error) — matches success criteria wording and V060 precedent
- Code: **V062** — next in the V06x series after V060 (OpaqueCallee) and V061 (OpaqueCalleeUnsafe)
- Mechanism: New `VcKind` variant or a `tracing::warn!` — follow the same pattern used for V060/V061 (new VcKind variant is preferred for consistency with existing diagnostic architecture)
- Message should reference both annotations: the `infer_summary` and `alias(p, q)` preconditions

### V06x code assignment
- V062 = "InferredSummaryWithAlias" — callee has both `is_inferred=true` and non-empty `alias_preconditions`
- Document in the same VcKind doc comment style as V060/V061

### Test placement
- Inline test in `vcgen.rs` test section near line 9324 (existing infer_summary tests for Phase 36-01)
- Test must: construct a callee summary with `is_inferred=true` + non-empty `alias_preconditions`, call `generate_call_site_vcs`, assert alias VCs are emitted AND no OpaqueCallee VCs are emitted

### Claude's Discretion
- Exact VcKind variant name (e.g., `InferredSummaryAlias` or `InferredWithAlias`)
- Whether the V062 diagnostic is a separate VC or a flag on the alias VC
- Exact wording of the diagnostic message body
