### Critical Integration Gap: ALIAS-01/ALIAS-02

**Location:** `crates/driver/src/callbacks.rs:443`

```rust
alias_preconditions: vec![],  // ← always empty, never populated
```

The HIR scanner reads `rust_fv::unsafe_requires::*` doc attributes into `contracts.requires` (the spec string) but contains no code to:
1. Tokenize the spec for `!alias(param_a, param_b)` patterns
2. Resolve parameter names to zero-based indices
3. Construct `AliasPrecondition { param_idx_a, param_idx_b }` entries

All alias VC injection code in `vcgen.rs:2447,2583–2630` has no input data and is permanently dead code.

**Why Phase 34 tests pass:** The tests exercise `vcgen.rs` alias VC generation in isolation, passing `alias_preconditions` directly to test functions — they do not test the full driver → HIR → callbacks → contract_db → vcgen pipeline.

**Fix:** Add `!alias(...)` parsing to the `callbacks.rs` HIR scanner where it processes `unsafe_requires` doc attributes. Parse `!alias(p, q)` predicates, look up parameter names in `fn_sig.inputs()`, resolve to indices, populate `alias_preconditions` before building `FunctionSummary`.
