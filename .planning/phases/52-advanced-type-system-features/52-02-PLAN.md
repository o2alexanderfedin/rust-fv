<task type="auto" tdd="true">
  <name>Task 2: Negative trait impls (!Send/!Sync) in TraitDatabase + ThreadSafety VCs</name>
  <files>crates/analysis/src/trait_analysis.rs, crates/analysis/src/vcgen.rs, crates/driver/src/callbacks.rs, crates/analysis/tests/negative_impl_tests.rs</files>
  <behavior>
    - Test 1: TraitDatabase registers negative impl !Send for a type
    - Test 2: Type with !Send transferred via thread spawn generates ThreadSafety VC (SAT = violation)
    - Test 3: Type with !Sync shared across threads generates ThreadSafety VC (SAT = violation)
    - Test 4: Type without negative impl does NOT generate extra ThreadSafety VCs
    - Test 5: SMT context includes NOT(is_send(MyType)) axiom for !Send types
  </behavior>
  <action>
1. Extend `TraitDatabase` in trait_analysis.rs with `negative_impls: HashMap<String, Vec<String>>` field. The key is trait name ("Send", "Sync"), value is list of types with negative impls.

2. Add methods to TraitDatabase:
   - `register_negative_impl(trait_name: &str, type_name: &str)` -- records the negative impl
   - `has_negative_impl(trait_name: &str, type_name: &str) -> bool` -- checks if type has negative impl for trait
   - `is_send(type_name: &str) -> Option<bool>` -- returns Some(false) if !Send, None if unknown
   - `is_sync(type_name: &str) -> Option<bool>` -- returns Some(false) if !Sync, None if unknown

3. In callbacks.rs, detect negative impls during HIR traversal. Negative impls in Rust are `impl !Trait for Type` -- look for `Impl` items where `polarity` is `ImplPolarity::Negative`. Extract the trait name and self type, call `db.register_negative_impl(trait_name, type_name)`.

4. In vcgen.rs, create `generate_negative_impl_vcs(func: &Function, db: &TraitDatabase) -> Vec<VerificationCondition>`:
   - For each thread spawn call site (reuse W080 detection), check if any argument type has `!Send` in the trait database
   - If so, generate a `DataRaceFreedom` VC with a SAT assertion (unconditional violation -- the type is definitively not thread-safe)
   - Similarly, for shared references across threads, check `!Sync`
   - Additionally, for any function that uses a type with !Send or !Sync, emit SMT axiom `(assert (not (is_send type_name)))` to strengthen assumptions

5. Wire `generate_negative_impl_vcs` into `generate_vcs_with_db`.

6. Initialize `negative_impls: HashMap::new()` in TraitDatabase::new().

7. Create `crates/analysis/tests/negative_impl_tests.rs` with 5+ tests per behavior list.

8. Run clippy and full test suite.
