  <name>Task 2: Wire IsTester into SwitchInt discriminant encoding and update E2E tests</name>
  <files>crates/analysis/src/vcgen.rs, crates/analysis/tests/e2e_verification.rs</files>
  <behavior>
    - Test: SwitchInt on enum discriminant generates IsTester branch conditions instead of Eq(discr, BitVecLit)
    - Test: Variant index 0 maps to first variant name, index 1 to second variant name, etc.
    - Test: Otherwise branch generates Not(Or(IsTester(v1), IsTester(v2), ...))
    - Test: SwitchInt on non-enum discriminant (Bool, Int) still uses Eq comparison (no regression)
    - Test: SetDiscriminant VCs use IsTester instead of uninterpreted function
    - Test: E2E test e2e_enum_datatype_match SMT output contains "((_ is mk-" tester syntax
    - Test: E2E test e2e_enum_datatype_match still produces UNSAT (correct verification)
  </behavior>
  <action>
    **Strategy:** The core change is in the SwitchInt terminator encoding. When the discriminant operand was assigned from `Rvalue::Discriminant(enum_place)`, we need to:
    1. Find the enum type of the discriminant source place
    2. Map the SwitchInt target value to the variant name
    3. Generate `Term::IsTester("mk-{variant_name}", Box::new(enum_place_term))` instead of `Term::Eq(discr_term, BitVecLit)`

    **Implementation steps:**

    1. **Add helper function `find_discriminant_source`** in vcgen.rs:
       Given a function and a discriminant operand local name, scan backward through basic block statements to find the `Statement::Assign(place, Rvalue::Discriminant(source_place))` that assigned to this local. Return `Some((source_local_name, enum_ty))` where `enum_ty` is from `find_local_type(func, source_local_name)` and matches `Ty::Enum(...)`.

    2. **Modify SwitchInt encoding** (vcgen.rs around line 1247-1312):
       Before the existing `for (value, target_block) in targets` loop, check if `discr` is a Copy/Move of a local that was assigned from `Rvalue::Discriminant`. If so:
       - Extract the enum type's variant list
       - For each target `(value, target_block)`, map `value` (as usize) to `variants[value as usize].0` (variant name)
       - Generate `Term::IsTester(format!("mk-{variant_name}"), Box::new(Term::Const(enum_source_local.clone())))` as the branch condition
       - For the otherwise branch, negate the disjunction of all explicit IsTester conditions
       If NOT an enum discriminant, fall through to existing Eq/BitVecLit logic.

    3. **Update `Rvalue::Discriminant` encoding** (vcgen.rs around line 1958):
       The discriminant assignment `_2 = discriminant(_1)` no longer needs to produce an uninterpreted function application. Since SwitchInt will now use IsTester directly on the enum source, the discriminant local itself becomes unused in the SMT encoding. Change the encoding to return `None` (no assertion needed for discriminant extraction -- the SwitchInt handles it via IsTester). This avoids declaring the uninterpreted `discriminant-{local}` function.

    4. **Update `encode_rvalue_for_switch`** (vcgen.rs around line 1694):
       The `Rvalue::Discriminant` arm here should also return `None` or be handled -- but since SwitchInt will now bypass this path for enum discriminants (using IsTester directly), this is safe.

    5. **Update `generate_set_discriminant_vcs`** (vcgen.rs around line 1471):
       Replace the uninterpreted function assertion with an IsTester assertion:
       - Look up the enum type from `find_local_type(func, &place.local)`
       - Map `variant_idx` to variant name from the Ty::Enum variants list
       - Generate `Term::IsTester(format!("mk-{variant_name}"), Box::new(Term::Const(place.local.clone())))` as the VC assertion

    6. **Remove the soundness note comment** at vcgen.rs:1966-1975 since native testers are now implemented.

    7. **Update E2E test `e2e_enum_datatype_match`** to assert:
       - SMT output contains `(_ is mk-` (tester syntax)
       - SMT output does NOT contain `discriminant-` (no uninterpreted function)
       - Result is still UNSAT (correct verification)

    8. **Add unit test `switchint_enum_discriminant_uses_is_tester`** that creates a function with enum discriminant + SwitchInt, generates VCs, and confirms IsTester appears in the output.

    9. Run `cargo clippy -p rust-fv-analysis` to check for warnings. Run `cargo test -p rust-fv-analysis` to confirm all tests pass including existing enum tests.
