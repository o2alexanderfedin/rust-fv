  <name>Task 1: Per-field partial struct move tracking (COMPL-14)</name>
  <files>crates/analysis/src/vcgen.rs, crates/analysis/src/ir.rs, crates/analysis/tests/partial_move_tests.rs</files>
  <behavior>
    - Test 1: Move of s.field_a marks (s, [Field(0)]) as moved; read of s.field_b (Field(1)) produces no VC
    - Test 2: Move of s.field_a then read of s.field_a generates UseAfterPartialMove VC
    - Test 3: Move of s.inner.x marks (s, [Field(0), Field(0)]) as moved; read of s.inner.y produces no VC
    - Test 4: Move of entire struct s marks all fields as moved; read of any field generates UseAfterPartialMove VC
    - Test 5: Re-assignment to a moved field clears the moved state; subsequent read produces no VC
    - Test 6: Function with no partial moves generates no UseAfterPartialMove VCs (backward compatible)
  </behavior>
  <action>
    1. In vcgen.rs, add a `FieldMoveTracker` struct:
       ```rust
       /// Tracks per-field move state for partial struct move analysis.
       struct FieldMoveTracker {
           /// Maps (local_name, projection_path) -> is_moved
           /// Projection path is Vec<usize> representing field indices at each level
           moved_fields: HashMap<(String, Vec<usize>), bool>,
       }
       ```

    2. Implement FieldMoveTracker methods:
       - `mark_moved(local: &str, projections: &[Projection])`: Extract field indices from projections, mark as moved
       - `mark_available(local: &str, projections: &[Projection])`: Clear moved state (for re-assignment)
       - `is_moved(local: &str, projections: &[Projection]) -> bool`: Check if field path is moved
       - For whole-struct moves (empty projection path), mark all known fields as moved

    3. Add `fn generate_partial_move_vcs(func: &Function) -> Vec<VerificationCondition>`:
       - Create FieldMoveTracker
       - Walk basic blocks sequentially
       - For each Statement::Assign with Rvalue::Use(Operand::Move(place)):
         - If place has Field projections: mark those fields as moved
         - If place has no projections (whole move): mark all fields of that local as moved
       - For each Statement::Assign where the destination has projections: mark_available (re-assignment clears move)
       - For each operand read (Operand::Copy or Operand::Move with Field projections):
         - Check if the read field path is_moved
         - If moved: generate UseAfterPartialMove VC with VcKind::UseAfterPartialMove
         - Description: "Field {path} of {local} used after partial move"

    4. Call generate_partial_move_vcs from generate_vcs, append results.

    5. In ir.rs, add a helper `Place::field_indices(&self) -> Vec<usize>` that extracts usize values from Projection::Field entries.

    6. Create test file with unit tests constructing Function IR with partial move patterns.
  </action>

  <name>Task 2: Disjoint field borrow splitting (COMPL-16)</name>
  <files>crates/analysis/src/borrow_conflict.rs, crates/analysis/tests/borrow_splitting_tests.rs</files>
  <behavior>
    - Test 1: &mut s.x (Field(0)) and &s.y (Field(1)) on same local are field-disjoint; no BorrowConflict VC
    - Test 2: &mut s.x (Field(0)) and &s.x (Field(0)) on same local are NOT disjoint; BorrowConflict VC generated
    - Test 3: &mut s.x (Field(0)) and &mut s.y (Field(1)) on same local are disjoint; no conflict
    - Test 4: &mut a.x and &b.y on different locals are already non-conflicting (existing behavior preserved)
    - Test 5: &mut s (no field projection) and &s.x conflict because whole-struct borrow overlaps any field
    - Test 6: Nested field disjointness: &mut s.inner.x (Field(0),Field(0)) and &s.inner.y (Field(0),Field(1)) are disjoint
  </behavior>
  <action>
    1. In borrow_conflict.rs, add `fn are_borrows_field_disjoint(borrow_a: &BorrowInfo, borrow_b: &BorrowInfo, func: &Function) -> bool`:
       - Extract the Place for each borrow from the function's statements (find the Rvalue::Ref that creates each borrow local)
       - If both borrows are on the same base local:
         - Extract Projection::Field indices from each Place
         - If either has no Field projections (whole-struct borrow): NOT disjoint
         - If both have Field projections: compare the first diverging field index; if different at any level, they are disjoint
       - If different base locals: return false (not applicable, handled by existing logic)

    2. Modify `detect_borrow_conflicts` to call `are_borrows_field_disjoint` before emitting a conflict:
       - If the shared and mutable borrows are on the same local AND are field-disjoint, skip the conflict
       - This is a refinement of the existing check, not a replacement

    3. Add field projection extraction helper: `fn extract_borrow_place(func: &Function, borrow_local: &str) -> Option<Place>`:
       - Walk statements to find the Assign that creates the borrow local (Rvalue::Ref)
       - Return the Place from the Ref

    4. Create test file with unit tests for field disjointness scenarios.
  </action>
