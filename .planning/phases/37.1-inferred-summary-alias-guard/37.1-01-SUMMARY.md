# Phase 37.1 Plan 01: Inferred Summary Alias Guard Summary

Guard is_inferred early-continue to emit alias VCs when callee has both is_inferred=true and non-empty alias_preconditions; wire V062 InferredSummaryAlias through diagnostics.rs and callbacks.rs following the V060/V061 pattern.

## What Was Implemented

### Guard Logic (vcgen.rs)

Modified `generate_call_site_vcs` at line 2442 to hoist `has_alias_preconditions` check before the `is_inferred` early-continue:

- **When is_inferred=true AND alias_preconditions empty:** `continue` (unchanged behavior — suppresses VC generation entirely)
- **When is_inferred=true AND alias_preconditions non-empty:** emit V062 `InferredSummaryAlias` always-SAT warning VC, then fall through to alias VC generation (do not `continue`)

The fix closes the ALIAS-01/ALIAS-02 edge-case gap where the unconditional `continue` at line 2441 silently dropped alias VCs when a callee had both `is_inferred=true` and `alias_preconditions`.

### VcKind::InferredSummaryAlias (vcgen.rs)

The variant was already present in the enum (lines 149-152) from the planner's context setup. No addition needed.

### diagnostics.rs Wiring

1. `vc_kind_description`: added arm returning `"inferred-summary callee with alias preconditions: alias VCs emitted (V062)"`
2. `suggest_fix`: added arm with V062 guidance (remove infer_summary or add explicit requires contracts)
3. Warning branch: added `|| failure.vc_kind == VcKind::InferredSummaryAlias` to report_kind condition

### callbacks.rs Wiring

1. `vc_kind_to_string`: added arm `VcKind::InferredSummaryAlias => "inferred_summary_alias"`
2. Failure-push filter: added `&& result.vc_location.vc_kind != rust_fv_analysis::vcgen::VcKind::InferredSummaryAlias` (SAT = diagnostic only, not a hard failure)

## Tests Added

Two new tests in the `// --- Phase 37.1: is_inferred + alias_preconditions co-occurrence tests ---` section of vcgen.rs:

1. `test_inferred_summary_with_alias_preconditions_emits_alias_vcs`: verifies InferredSummaryAlias VC present, OpaqueCallee absent when is_inferred=true + alias_preconditions non-empty
2. `test_inferred_summary_without_alias_preconditions_unchanged`: regression guard — is_inferred=true + empty alias_preconditions emits ZERO call-site VCs
