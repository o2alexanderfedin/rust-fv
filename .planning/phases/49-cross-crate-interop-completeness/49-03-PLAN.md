   - Add a new function `compose_adapter_contracts(adapters: &[(String, &StdlibContract)]) -> Contracts` that takes an ordered list of adapter names + their contracts and produces a composed contract.
   - Composition rules:
     - Length contracts chain: if filter says `len <= original` and map says `len == input_len`, composed result is `len <= original`
     - Element contracts chain: filter's predicate becomes a precondition on map's input elements
     - The composed postcondition should reference the original iterator's properties

   - When processing a call to an iterator adapter method (map, filter, take, etc.), check if the receiver is itself the result of another adapter call (chain detection).

   - `register_map`: add element postcondition `forall(|i| result.element(i) == f(source.element(i)))`
