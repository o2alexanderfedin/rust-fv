    - Test: Non-HRTB function with regular lifetime params is unchanged (regression)
    - Test: Multiple HRTB bounds (`for<'a, 'b>`) produce correct multi-variable quantification

       - `param_tys: Vec<Ty>` — the parameter types (may reference quantified lifetimes)

       - When extracting GenericParamDefKind::Type bounds, check for `for<'a>` binders
       - rustc represents these as `ty::Binder<...>` with bound regions
       - Extract quantified lifetime names and the inner trait bound
       - Store as `hrtb_bounds: Vec<HrtbBound>` on GenericParam (or Function)

       - For each HRTB bound, emit a universally quantified SMT assertion
       - The lifetime parameter becomes a fresh SMT variable
       - The closure/fn spec is wrapped in `(forall ((lifetime_a Int)) ...spec...)`
       - Lifetime erasure: since lifetime is a region, encode as an abstract sort or Int (region identifier)
       - The contract applies for ALL lifetime instantiations (soundness: universal quantification)

    4. In encode_term.rs, handle lifetime-parameterized reference types:
       - When a type references a quantified lifetime, encode the reference as parameterized by the lifetime variable
       - For the purpose of verification, lifetime-parameterized refs behave like regular refs (Rust's borrow checker already ensures correctness)

    5. Write hrtb_tests.rs with unit tests.

<task type="auto" tdd="true">
  <name>Task 2: Union types — ghost state, reinterpretation VCs, MIR detection</name>
  <files>crates/analysis/src/vcgen.rs, crates/analysis/src/encode_term.rs, crates/driver/src/mir_converter.rs, crates/analysis/tests/union_tests.rs</files>
  <behavior>
    - Test: MIR converter detects union ADT and creates Ty::Union with correct field list
    - Test: Writing to a union field sets active_field ghost variable to that field's index
    - Test: Reading the active field generates no safety VC (just reinterpretation encoding)
    - Test: Reading an inactive field generates UnionAccess VC (V130) — Z3 reports SAT (violation)
    - Test: Union stored as max-size bitvector; field reads extract appropriate bits
    - Test: repr(C) union has defined field offsets per C ABI layout
  </behavior>
  <action>
    1. In mir_converter.rs, detect union ADTs:
       - Check `adt_def.is_union()` in the MIR type conversion
       - Map union fields to `Ty::Union(name, fields)` with field names and types
       - Detect `#[repr(C)]` attribute for layout-defined unions

    2. In vcgen.rs, implement union ghost state tracking:
       - At function entry, for each union local, declare ghost variable `{local}_active_field: Int` initialized to -1 (uninitialized)
       - On union field write (`place.field = value`), set `{local}_active_field = field_index`
       - On union field read (`let x = place.field`):
         a. Generate `UnionAccess` VC: assert `{local}_active_field == field_index` (reading active field is safe)
         b. If field types differ, generate reinterpretation-cast encoding (bitwise equivalence)

    3. In encode_term.rs, implement union field encoding:
       - Union stored as BitVec(max_field_size * 8) — same pattern as transmute
       - Field write: store entire bitvector (zero-extend smaller fields)
       - Field read: extract(offset, field_size) from the union bitvector
       - Reinterpretation: if reading a different type than written, the extracted bits are reinterpreted as the target type (same as transmute encoding from Phase 50)

    4. For repr(C) unions:
       - Use `mem::size_of` and `mem::align_of` SMT constants from Phase 50
       - Assert exact field offsets (all at offset 0 for unions, but sizes differ)
       - Layout assertions in SMT preamble

    5. Populate `func.union_ghost_states` in MIR converter when union locals are detected.

    6. Write union_tests.rs covering active field tracking, reinterpretation VCs, and repr(C) layout.
  </action>
